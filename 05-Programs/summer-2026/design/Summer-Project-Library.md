*Six pre-designed project briefs with personalization layers — ages 10–12*

## How This Library Works
*Two layers per project — fixed core and personalization layer*

Every project has two layers.

**Fixed core:** The concept, deliverable, roles, and Saturday showcase format. These do not change per student. They are designed to be irresistible on their own — before any personalization is applied.

**Personalization layer:** Used by the PBL Project Designer skill. Contains the match profile, driving question variants, role-to-signal guidance, and diagnostic moments. The skill applies this layer to the student's data to produce a personalized version of each project.

**Diagnostic moments:** Every project has two built-in observation points. These are invisible to the student. The student is just doing the project. The Guide is watching.
- **Confirmation moment:** A task designed to surface a signal already in the element hypothesis. Confirms and deepens what the Guide already suspects.
- **Discovery moment:** A task designed to expose the student to territory outside their known pull. May surface something neither the student nor the Guide expected.

## Friction Points
*Known operational risks to resolve before launch*

**The Food Business:** Confirm kitchen access, food safety protocol, and ingredients cost before the brief goes live. California health code treats a selling event and a tasting event differently. Decide which this is before enrollment opens.

**The Science Tool:** Students need a topic narrow enough to explain in one tool. Guides must coach topic scoping on Monday. "How the universe works" cannot become an app in five days. "How black holes form" can.
